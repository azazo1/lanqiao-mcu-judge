# NA16 综合题 - 多源环境巡检终端

这个 sample 不再沿用原来的液位题, 而是整理成一道更偏向底层综合能力的训练题. 目标不是堆功能, 而是让 `EEPROM`, `DS18B20`, `DS1302`, `矩阵按键`, `数码管`, `LED`, `PCF8591`, `UART1`, `超声波`, `NE555` 这些常见模块之间形成一套有状态, 可保存, 可联动, 可判分的完整系统.

## 题目目标

编写一个 "多源环境巡检终端". 终端需要完成下面 4 类工作:

- 维护时钟和日期, 支持 `24h` 和 `12h` 两种显示.
- 周期采集温度, 光敏电阻, 滑动变阻器, 超声波距离, NE555 频率.
- 通过按键和 `UART1` 调整工作参数, 并在双键长按后保存到 `EEPROM`.
- 通过数码管和 `LED` 实时显示当前页面和告警状态.

题目重点是驱动正确性, 时序稳定性, 状态机设计, 非易失数据管理, 以及串口复杂格式解析.

## EEPROM 约定

程序只使用 `AT24C02` 的 `0x00..0x05` 这 6 个地址.

| 地址 | 含义 | 默认值 |
| --- | --- | --- |
| `0x00` | 锁字节 0 | `0xA5` |
| `0x01` | 锁字节 1 | `0x5A` |
| `0x02` | 配置字节 | `0x00` |
| `0x03` | 音速配置, 实际值为 `300 + EEPROM[0x03]` | `40` |
| `0x04` | DAC 输出码值 | `128` |
| `0x05` | 校验和 | `0xA7` |

`EEPROM[0x02]` 的位定义如下:

- bit0: `0` 表示 `24h`, `1` 表示 `12h`
- bit2..bit1: `DS18B20` 精度等级, 范围 `0..3`
- bit7..bit3: 保留

校验和规则:

```text
checksum = (byte0 + byte1 + byte2 + byte3 + byte4) & 0xFF
```

上电后必须先检查 `0x00`, `0x01`, `0x05` 是否有效.

- 如果锁字节或校验和错误, 说明这是首次上电或 `EEPROM` 数据无效. 程序必须装载默认参数, 并把默认参数重新写回 `EEPROM`.
- 如果锁字节和校验和都正确, 程序必须按 `EEPROM` 中的数据恢复 `12h/24h`, 温度精度, 音速配置和 DAC 输出.

双键长按保存时, 也必须同步更新 `0x05`.

## 传感器与采样要求

### DS1302

- 需要读取日期和时间.
- 需要支持写入日期和时间.
- 需要支持 `24h -> 12h` 和 `12h -> 24h` 显示转换.
- 评测会覆盖 `23 -> 11 PM`, `00 -> 12 AM`, `12 -> 12 PM` 这几个边界.

### DS18B20

- 需要读取当前温度.
- 需要支持精度等级 `0..3`, 对应 `9bit`, `10bit`, `11bit`, `12bit`.
- 精度切换不能只改 RAM 变量, 需要真正写入器件配置并完成非易失保存动作.
- 评测会重点检查 `25.9375` 在 4 个精度等级下的量化结果.

### PCF8591

- `AIN1` 接光敏电阻, `AIN3` 接滑动变阻器.
- 程序需要周期读取这两个通道的 ADC 值.
- 程序需要输出 DAC 码值到 `AO`.
- DAC 码值不直接显示到数码管, 但会通过 `L6` 的 PWM 占空比和 `UART1` 查询结果体现出来.

### 超声波

- 需要测量距离, 页面显示单位为 `cm`.
- 距离计算使用当前配置音速.
- 默认音速是 `340m/s`, 对应 `EEPROM[0x03] = 40`.
- 允许通过 `UART1` 把音速改到 `300..399m/s`.

### NE555

- 需要通过 `SIG_OUT -> P3.4/T0` 这条链路测频.
- 评测时会显式安装跳帽 `NET_SIG <-> SIG_OUT`.
- 页面显示单位为 `Hz`.
- 频率建议使用固定窗口统计, 评测按 `1s` 左右测频窗口设计.

## 数码管页面

系统共有 4 个页面, 上电默认停在第 0 页.

### 第 0 页, 时钟页

该页有 2 个子页面:

- 时间页显示 `HH-MM-SS`
- 日期页显示 `YY-MM-DD`

上电默认显示时间页.

### 第 1 页, 温度页

显示格式:

```text
TT.TTT-P
```

示例:

- `25.500-0`
- `25.937-3`
- `-5.125-3`

说明:

- 左侧是温度值.
- 最后一位 `P` 是精度等级 `0..3`.
- 评测显示范围只覆盖 `-9.999..99.999`.

### 第 2 页, AD 页

显示格式:

```text
AAA-BBB
```

- `AAA` 是 `AIN1` 的 ADC 码值, 范围 `000..255`
- `BBB` 是 `AIN3` 的 ADC 码值, 范围 `000..255`

DAC 调节不允许改写这个页面的显示内容.

### 第 3 页, 距离频率页

显示格式:

```text
DDD-FFFF
```

- `DDD` 是按当前音速计算出的距离值, 单位 `cm`, 范围 `000..999`
- `FFFF` 是当前测得的频率值, 单位 `Hz`, 范围 `0000..9999`

## LED 规则

- `L1` `L2` `L3` `L4` 分别指示当前页 `0` `1` `2` `3`, 同时只允许 1 个点亮.
- `L5` 为心跳灯, 以 `1Hz` 方波闪烁.
- `L6` 为 DAC 占空比指示灯, 使用 PWM 驱动, 占空比与当前 DAC 码值成正比.
- `L7` 为距离告警灯. 当距离 `< 20cm` 或 `> 80cm` 时, 以 `2Hz` 闪烁.
- `L8` 为频率告警灯. 当频率 `< 800Hz` 或 `> 1800Hz` 时, 以 `5Hz` 闪烁.

## 按键规则

使用矩阵键盘模式.

- `S4` 单击短按: 页面按 `0 -> 1 -> 2 -> 3 -> 0` 循环切换.
- `S5` 单击短按: 仅在第 0 页有效, 在 "时间页" 和 "日期页" 之间切换.
- `S6` 双击: 仅在第 0 页有效, 切换 `24h` 和 `12h`.
- `S7` 单击短按:
  - 在第 1 页, 精度等级 `+1`, 超过 `3` 后回到 `0`
  - 在第 2 页, DAC 码值 `+16`, 上限饱和到 `255`
- `S8` 单击短按:
  - 在第 1 页, 精度等级 `-1`, 低于 `0` 后回到 `3`
  - 在第 2 页, DAC 码值 `-16`, 下限饱和到 `0`
- `S8 + S9` 双键长按超过 `2s`: 保存当前 `12h/24h`, 精度等级, 音速配置和 DAC 码值到 `EEPROM`

`S9` 单独短按没有业务功能, 只参与保存组合键.

## UART1 协议

串口协议使用 ASCII 文本帧:

- 帧头固定为 `@`
- 帧尾固定为 `#`
- 字段分隔符是 `,`
- 键值对分隔符是 `=`

### 查询配置

```text
@Q,CFG#
```

返回:

```text
@CFG,<mode>,<pre>,<spd>,<dac>#
```

其中:

- `<mode>` 取值 `12` 或 `24`
- `<pre>` 范围 `0..3`
- `<spd>` 范围 `300..399`
- `<dac>` 范围 `0..255`

### 查询全部状态

```text
@Q,ALL#
```

返回:

```text
@ALL,<date>,<time>,<mode>,<pre>,<spd>,<dac>,<ain1>,<ain3>,<dist>,<freq>#
```

返回字段约定:

- `<date>` 固定格式 `YY-MM-DD`
- `<time>` 固定格式 `HH-MM-SS`, 按当前显示模式返回
- `<ain1>` `<ain3>` `<dist>` 使用 3 位十进制, 不足补 `0`
- `<freq>` 使用 4 位十进制, 不足补 `0`

### 修改配置

```text
@CFG,<field>=<value>[,<field>=<value>...]#
```

允许字段:

- `MODE=12` 或 `MODE=24`
- `PRE=0..3`
- `SPD=300..399`
- `DAC=0..255`

要求:

- 字段顺序可以任意
- 允许只发其中一部分字段
- 收到合法配置后返回 `@OK,CFG#`
- 帧格式错误返回 `@ERR,FRAME#`
- 数值越界或字段不支持返回 `@ERR,VALUE#`

### 修改 RTC

```text
@RTC,DATE=YY-MM-DD,TIME=HH:MM:SS#
```

要求:

- 时间输入按 `24h` 语义解释
- 日期和时间都合法时返回 `@OK,RTC#`
- 任一字段非法时返回 `@ERR,VALUE#`

## 实现建议

- `DS18B20` 精度等级可以直接映射到配置寄存器 `R1:R0`.
- `UART1` 建议做成 "接收缓冲 + 超时定帧 + 命令分发" 的结构, 不要在中断里直接做复杂解析.
- `EEPROM` 建议封装成 `load_config()`, `save_config()`, `config_checksum()` 3 个函数.
- 第 3 页距离显示建议和频率显示分时更新到同一组缓存, 不要把测距和测频逻辑直接写进数码管扫描中断.
- `L6` 的 PWM 频率建议固定在约 `1kHz`.

## 测试设计

这组 judge 重点覆盖 6 类需求:

- 首次上电数据锁校验, 默认值回写, 有效配置恢复
- `DS1302` 时间显示, 日期显示, `12h/24h` 转换
- `DS18B20` 温度读取, 精度切换, 负温显示
- `PCF8591` 双通道 ADC, DAC 输出, `L6` PWM 占空比
- 超声波音速配置, NE555 测频, `L7` `L8` 告警闪烁
- `UART1` 复杂帧解析, 查询, 修改, 非法输入处理

脚本划分如下:

- `judge/smoke.rhai`
  - 覆盖上电默认状态, 4 个页面的基本显示, `EEPROM` 默认值回写
- `judge/key_and_save.rhai`
  - 覆盖单键短按, 双击, 双键长按保存, `DS18B20` 精度切换, DAC 调节与恢复
- `judge/sensor_and_alarm.rhai`
  - 覆盖 ADC 显示, 音速改动后的测距, 测频显示, `L7` `L8` 闪烁
- `judge/uart_protocol.rhai`
  - 覆盖 `UART1` 查询, 复杂配置帧, RTC 设置帧, 非法帧处理

## 评测原则

- judge 只对关键行为断言, 不写大段基于文案的脆弱测试.
- 对 `ADC`, 距离, 频率, 占空比这类量测结果保留合理余量, 不做不必要的严格相等比较.
- 对 `UART1` 的配置查询和错误返回做精确断言, 让失败点可以直接定位到解析逻辑.
- 对需要跨上电验证的项目, 优先使用 `EEPROM` 字节和 `POWER_RESET` 做直接检查, 避免只通过数码管反推内部状态.
