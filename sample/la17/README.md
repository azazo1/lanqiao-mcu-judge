# LA17 综合题 - 育苗舱联控终端

这个 sample 参考 `na16` 的综合题强度, 但题型重新设计成 "育苗舱联控终端". 目标不是复刻上一题, 而是换一个业务场景, 用另一套状态组织和协议设计, 去考察同一批底层能力点.

系统需要同时管理时钟, 温度, 光照, 手动光控, 液位距离, 风机频率, 参数掉电保存和串口配置. 重点在驱动正确性, 状态机设计, 非易失参数管理, 和复杂串口帧解析.

## 题目背景

某育苗舱控制板需要实现下面功能:

- 通过 `DS1302` 维护日期和时间, 支持 `24h` 和 `12h` 显示.
- 通过 `DS18B20` 读取育苗舱温度, 并支持切换温度精度等级.
- 通过 `PCF8591` 读取光敏电阻 `AIN1` 和滑动变阻器 `AIN3`.
- 根据光照和工作模式输出 DAC 控制电压, 并用 `L6` 的 PWM 占空比表示当前 DAC 输出强度.
- 通过超声波测量营养液液面距离, 并允许修改音速参数.
- 通过 `NE555` 频率信号读取风机转速脉冲频率.
- 通过按键完成页面切换, 模式切换, 参数修改和掉电保存.
- 通过 `UART1` 完成查询, 配置修改和 RTC 设置.

## EEPROM 约定

程序只使用 `AT24C02` 的 `0x00..0x05` 这 6 个地址.

| 地址 | 含义 | 默认值 |
| --- | --- | --- |
| `0x00` | 锁字节 0 | `0x16` |
| `0x01` | 锁字节 1 | `0x71` |
| `0x02` | 配置字节 | `0x00` |
| `0x03` | 音速偏移, 实际音速为 `320 + EEPROM[0x03]` | `20` |
| `0x04` | 手动光控 DAC 码值 | `96` |
| `0x05` | 校验字节 | `0x49` |

`EEPROM[0x02]` 的位定义如下:

- bit0: `0` 表示 `24h`, `1` 表示 `12h`
- bit2..bit1: `DS18B20` 精度等级, 范围 `0..3`
- bit3: 光控模式, `0` 自动, `1` 手动
- bit7..bit4: 保留

校验规则:

```text
checksum = byte0 ^ byte1 ^ byte2 ^ byte3 ^ byte4 ^ 0x5A
```

上电后先检查锁字节和校验字节:

- 如果任一项不合法, 视为首次上电或数据损坏, 程序必须装载默认参数并回写到 `EEPROM`.
- 如果数据合法, 程序必须按 `EEPROM` 恢复 `12h/24h`, 温度精度, 光控模式, 音速, 手动 DAC 值.

## 页面设计

系统共有 4 个主页面, 上电默认停在第 0 页.

### 第 0 页, 时钟页

有 2 个子页面:

- 时间页显示 `HH-MM-SS`
- 日期页显示 `YY-MM-DD`

### 第 1 页, 温度页

显示格式:

```text
TT.TTT-P
```

- 左侧显示当前温度
- 最后一位 `P` 显示温度精度等级 `0..3`

例如:

- `25.500-0`
- `25.750-1`
- `-5.125-3`

### 第 2 页, 光控页

有 2 个子页面:

- ADC 子页面显示 `AAA-BBB`
  - `AAA` 是 `AIN1` 的 ADC 码值
  - `BBB` 是 `AIN3` 的 ADC 码值
- DAC 子页面显示 `M-CCC`
  - `M` 为光控模式, `0` 自动, `1` 手动
  - `CCC` 为当前有效 DAC 输出码值

### 第 3 页, 液位风机页

显示格式:

```text
DDD-FFFF
```

- `DDD` 是按当前音速计算出的液面距离值, 单位 `cm`
- `FFFF` 是当前测得的风机频率值, 单位 `Hz`

## 光控规则

光控有 2 种模式:

- 自动模式
  - 若 `AIN1 < AIN3`, 说明环境光低于旋钮阈值, 则 DAC 输出固定为 `224`
  - 否则 DAC 输出固定为 `32`
- 手动模式
  - DAC 输出固定为保存的手动 DAC 值

`L6` 使用 PWM 显示当前有效 DAC 输出强度, 占空比与 DAC 码值成正比.

## LED 规则

- `L1` `L2` `L3` `L4` 分别指示当前页 `0` `1` `2` `3`, 同时只允许 1 个点亮
- `L5` 为心跳灯, 以 `1Hz` 闪烁
- `L6` 为 DAC 强度指示灯, 使用 PWM 输出
- `L7` 为液位告警灯, 当距离 `< 15cm` 或 `> 70cm` 时以 `2Hz` 闪烁
- `L8` 为风机告警灯, 当频率 `< 1000Hz` 或 `> 2000Hz` 时以 `4Hz` 闪烁

## 按键规则

使用矩阵键盘模式.

- `S4` 单键短按
  - 主页面按 `0 -> 1 -> 2 -> 3 -> 0` 循环切换
- `S5` 单键短按
  - 在第 0 页切换时间页和日期页
  - 在第 2 页切换 ADC 子页面和 DAC 子页面
- `S6` 双击
  - 在第 0 页切换 `24h` 和 `12h`
  - 在第 2 页切换自动光控和手动光控
- `S7` 单键短按
  - 在第 1 页使温度精度 `+1`, 超过 `3` 后回到 `0`
  - 在第 2 页的 DAC 子页面且处于手动模式时, 手动 DAC 值 `+8`, 上限饱和到 `255`
- `S8` 单键短按
  - 在第 1 页使温度精度 `-1`, 低于 `0` 后回到 `3`
  - 在第 2 页的 DAC 子页面且处于手动模式时, 手动 DAC 值 `-8`, 下限饱和到 `0`
- `S8 + S9` 双键长按超过 `2s`
  - 将当前 `12h/24h`, 温度精度, 光控模式, 音速参数, 手动 DAC 值保存到 `EEPROM`

`S9` 单独短按没有业务功能, 只参与保存组合键.

## 超声波和频率要求

- 超声波距离使用当前配置音速计算
- 音速允许设置到 `320..379m/s`
- `NE555` 频率通过 `SIG_OUT -> P3.4/T0` 链路输入
- 评测时会显式安装跳帽 `NET_SIG <-> SIG_OUT`

## UART1 协议

协议采用文本帧:

- 帧头固定为 `$`
- 帧尾固定为 `;`
- 分段分隔符为 `|`
- 键值分隔符为 `:`

### 查询配置

```text
$Q|CFG;
```

返回:

```text
$CFG|CLK:24|TP:0|LM:0|SV:340|DAC:096;
```

字段含义:

- `CLK`: `12` 或 `24`
- `TP`: 温度精度等级 `0..3`
- `LM`: 光控模式, `0` 自动, `1` 手动
- `SV`: 当前音速
- `DAC`: 手动 DAC 保存值, 使用 3 位十进制

### 查询环境量

```text
$Q|ENV;
```

返回:

```text
$ENV|TMP:+25.500|L:203|VR:051|DIST:035|FREQ:1200;
```

字段要求:

- `TMP` 带符号, 并按当前温度精度量化后返回
- `L` 表示 `AIN1`
- `VR` 表示 `AIN3`
- `DIST` 为 3 位十进制
- `FREQ` 为 4 位十进制

### 修改配置

```text
$SET|CLK:12|TP:1|LM:1|SV:360|DAC:144;
```

要求:

- 字段顺序允许任意
- 允许只提供部分字段
- 所有字段都合法时返回 `$OK|SET;`
- 帧格式错误返回 `$ERR|FRAME;`
- 字段非法或数值越界返回 `$ERR|VALUE;`

### 设置 RTC

```text
$RTC|DATE:26-12-31|TIME:23:58:50;
```

要求:

- 日期和时间都合法时返回 `$OK|RTC;`
- 任一字段非法时返回 `$ERR|VALUE;`
- 时间输入总是按 `24h` 语义解释

## 实现建议

- `DS18B20` 精度等级可以直接映射到配置寄存器 `R1:R0`
- `UART1` 建议采用 "接收缓冲 + 超时定帧 + 解析分发" 结构
- `EEPROM` 建议封装成 `load_params()`, `save_params()`, `calc_checksum()` 3 个函数
- 光控模式判断和 DAC 更新建议独立于数码管扫描逻辑
- `L6` 的 PWM 频率建议固定在约 `1kHz`

## 测试设计

judge 不按 "堆一串操作后只看最终结果" 的方式写, 而是按软件测试里的覆盖准则拆分:

- 状态迁移覆盖
  - 首次上电默认恢复
  - 有效 `EEPROM` 恢复
  - `24h <-> 12h`
  - 自动光控 <-> 手动光控
  - 页面切换和子页面切换
- 边界值分析
  - 小时 `00`, `12`, `23`
  - 温度精度 `0`, `3`
  - 手动 DAC `0`, `255`
  - 距离告警阈值 `15`, `70`
  - 频率告警阈值 `1000`, `2000`
  - 音速范围 `320`, `379`
- 等价类划分
  - 合法锁字节/非法锁字节
  - 合法校验/非法校验
  - 自动光控亮环境, 暗环境, 临界相等环境
  - 串口合法帧, 帧格式错误, 字段错误, 数值越界
- 判定表覆盖
  - 光控判定 `AIN1 < AIN3` 与其他情况
  - 距离告警和频率告警的正常/异常分支
  - 串口多字段部分更新时 "只改一个字段, 其他字段保持不变" 的判定

脚本划分如下:

- `judge/smoke.rhai`
  - 覆盖首次上电锁校验失败后的默认值回写
  - 覆盖 4 个页面和关键子页面的基本显示
- `judge/key_and_save.rhai`
  - 覆盖 `00`, `12`, `23` 三个小时边界下的 `24h/12h` 转换
  - 覆盖温度精度 `0 -> 1 -> 2 -> 3 -> 0 -> 3`
  - 覆盖自动光控下按键无效, 手动光控下 DAC 调节, 保存和掉电恢复
- `judge/sensor_and_alarm.rhai`
  - 覆盖自动光控的亮环境, 暗环境, 相等阈值环境
  - 覆盖音速变更后的测距结果
  - 覆盖距离和频率告警的阈值内, 阈值点, 阈值外三类情况
- `judge/uart_protocol.rhai`
  - 覆盖配置查询, 环境量查询, 多字段乱序设置, 部分字段更新
  - 覆盖 `DAC` 的 `000` 和 `255` 边界值
  - 覆盖非法数值, 非法字段, 非法日期, 非法帧格式

## 评测原则

- judge 只断言关键行为, 不写冗余的文案型测试
- 对 ADC, 距离, 频率, PWM 占空比保留合理余量
- 对 `UART1` 返回格式做精确断言, 便于直接定位协议解析问题
- 对掉电保存相关行为, 优先通过 `EEPROM` 原始字节和 `POWER_RESET` 验证
