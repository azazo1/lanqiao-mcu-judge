# Persistent State

这个样例专门用于验证评测器新增的 `export_persistent_state()` , `load_persistent_state(...)` 和 `reset()` 接口.

建议做成独立按键题, 使用 `BTN` 模式.

## 显示要求

8 位数码管固定显示:

```text
VV-R-EEE
```

- `VV` 是 2 位十进制会话计数值, 范围 `00..99`.
- 第 3 位固定显示 `-`.
- `R` 是 `DS18B20` 当前精度等级, 范围 `0..3`.
- 第 5 位固定显示 `-`.
- `EEE` 是 `AT24C02` 指定地址中的 3 位十进制数值, 范围 `000..255`.

例如:

- `00-3-000`
- `02-0-034`
- `15-2-255`

## 功能要求

上电后:

- 从 `DS18B20` 读取当前精度等级并显示到 `R`.
- 从 `AT24C02` 的 `0x10` 地址读取 1 字节数值并显示到 `EEE`.
- `VV` 清零为 `00`.

按键行为:

- `S4`: `VV = (VV + 1) % 100`. 这个值只保存在 RAM 中, 不写入任何外设.
- `S5`: `R = (R + 3) % 4`. 修改 `DS18B20` 精度等级, 并立即写入器件的非易失区域.
- `S6`: `R = (R + 1) % 4`. 修改 `DS18B20` 精度等级, 并立即写入器件的非易失区域.
- `S7`: `EEE = (EEE + 256 - 17) % 256`. 将新值立即写入 `AT24C02[0x10]`.
- `S8`: `EEE = (EEE + 17) % 256`. 将新值立即写入 `AT24C02[0x10]`.

显示更新要求:

- 每次按键生效后, 数码管显示应及时刷新.
- `VV` 的变化不能影响 `R` 和 `EEE`.
- `R` 和 `EEE` 的变化不能依赖重新上电才能显示出来.

## 评测重点

后续评测会重点检查下面这些行为:

- 修改 `R` 和 `EEE` 后, `export_persistent_state()` 导出的字符串会发生对应变化.
- 调用 `reset()` 后, `VV` 会回到 `00`, 但 `R` 和 `EEE` 会保留最近一次写入后的值.
- 在运行过程中先导出一个旧状态字符串, 再继续修改 `R` 和 `EEE`, 最后调用 `load_persistent_state(...)` 并配合 `reset()` , 可以恢复到更早导出的那组非易失状态.

## 实现建议

- `DS18B20` 精度等级建议直接对应 `9bit, 10bit, 11bit, 12bit`.
- `AT24C02` 建议固定只用一个地址, 不需要做分页显示.
- 评测时不会要求你把持久状态字符串显示到数码管上, 固件只需要正确驱动外设并显示当前结果.
