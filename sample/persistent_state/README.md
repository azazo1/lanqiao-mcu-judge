# Persistent State

这个样例专门用于验证评测器新增的 `export_persistent_state()`, `load_persistent_state(...)` 和 `reset()` / `reset(POWER_RESET)` 接口.

建议做成矩阵键盘题, 使用默认的 `KEYBOARD` 模式.

## 显示要求

8 位数码管固定显示:

```text
VV-R-EEE
```

- `VV` 是 2 位十进制会话计数值, 范围 `00..99`.
- 第 3 位固定显示 `-`.
- `R` 是 `DS18B20` 当前精度等级, 范围 `0..3`.
- 第 5 位固定显示 `-`.
- `EEE` 是当前 EEPROM `cursor` 所指地址中的 3 位十进制数值, 范围 `000..255`.

例如:

- `00-0-000`
- `02-3-034`
- `15-1-255`

## EEPROM 规则

- `cursor` 是一个运行时地址指针.
- `cursor` 的有效范围固定为 `0x00..=0x20`.
- 程序需要支持对 `0x00..=0x20` 这一个 EEPROM 块做顺序扫描.
- `EEE` 始终显示 `AT24C02[cursor]` 的值.

上电后:

- 从 `DS18B20` 读取当前精度等级并显示到 `R`.
- `cursor` 初始化为 `0x00`.
- `EEE` 显示 `AT24C02[0x00]`.
- `VV` 清零为 `00`.

## 按键行为

- `S4`: `VV = (VV + 1) % 100`. 这个值只保存在 RAM 中, 不写入任何外设.
- `S5`: `R = (R + 3) % 4`. 修改 `DS18B20` 精度等级, 并通过 `Copy Scratchpad` 真正写入器件的非易失区域.
- `S6`: `R = (R + 1) % 4`. 修改 `DS18B20` 精度等级, 并通过 `Copy Scratchpad` 真正写入器件的非易失区域.
- `S7`: `cursor` 向前移动 1 格. 如果当前已经在 `0x00`, 则回绕到 `0x20`, 移动之后读取当前 `EEE`.
- `S8`: `cursor` 向后移动 1 格. 如果当前已经在 `0x20`, 则回绕到 `0x00`, 移动之后读取当前 `EEE`.
- `S9`: `AT24C02[cursor]` 的值 -= 1, 然后写回.
- `S10`: `AT24C02[cursor]` 的值 += 1, 然后写回.

## 显示更新要求

- 每次按键生效后, 数码管显示应及时刷新.
- `VV` 的变化不能影响 `R` 和 `EEE`.
- `R` 的变化不能影响 `VV` 和 `EEE`.
- `cursor` 变化后, `EEE` 必须立即切换到新地址对应的值.

## 评测重点

后续评测会重点检查下面这些行为:

- 修改 `R` 后, `export_persistent_state()` 导出的字符串会发生对应变化.
- 修改 EEPROM 块中某些地址后, `export_persistent_state()` 导出的字符串会发生对应变化.
- 调用 `reset()` 或 `reset(POWER_RESET)` 后, `VV` 会回到 `00`.
- 调用 `reset()` 或 `reset(POWER_RESET)` 后, `R` 和 EEPROM 中已经写入的内容会保留.
- 调用 `reset()` 或 `reset(POWER_RESET)` 后, `cursor` 会重新回到 `0x00`, 因而 `EEE` 应重新显示 `AT24C02[0x00]`.
- 在运行过程中先导出一个旧状态字符串, 再继续修改 `R` 和 EEPROM 内容, 最后调用 `load_persistent_state(...)` 并配合 `reset()` 或 `reset(POWER_RESET)`, 可以恢复到更早导出的那组非易失状态.
- 程序必须能够通过 `S7` 和 `S8` 顺序扫描 `0x00..=0x20` 整个块, 不能把 `cursor` 走出这个范围.
- 评测需要读取 AT24C02 中存储的数据进行额外的比较, 而不仅仅是通过 seg 来判断.

## 实现建议

- `DS18B20` 精度等级建议直接对应 `9bit`, `10bit`, `11bit`, `12bit`.
- `AT24C02` 建议直接把 `0x00..=0x20` 当作一个可浏览, 可写入的测试块.
- 评测时不会要求你把持久状态字符串显示到数码管上, 固件只需要正确驱动外设并显示当前结果.
